Service rules phải khóa9.1. Source zonekhông cho tạo trùng zone_code không cho deactivate nếu policy cấm khi đang còn source origin active 9.2. Source originphải bind đúng source_zone_id không cho create origin nếu zone không tồn tại hoặc inactive verification phải là record riêng, không overwrite bừa source origin evidence là record riêng, không nhét blob vào record chính 9.3. IdempotencyCác command sau phải đi qua idempotency_registry:create zone create source origin attach evidence verify source origin Đây là rule đã khóa ở foundation layer và có nhắc rõ trong CODE 01.10. SourceOriginControllerAction tối thiểu:list create detail attachEvidence verify 10.3. SourceZoneQueryServicelist zones get zone detail nếu cần 10.4. SourceZoneCommandServicecreate zone update zone deactivate zone 10.5. SourceOriginQueryServicelist origins get detail origin 10.6. SourceOriginCommandServicecreate origin attach evidence verify origin 10.7. SourceOriginValidationServicevalidate zone exists validate zone active validate unique source origin code validate evidence target validate verification request 11. Source origin verification screenPhải có tối thiểu:danh sách source origin filter theo zone / verify status xem evidence verify source origin xem lịch sử verification 12. Test phải tạo file nào12.1. tests/integration/source-origin/SourceZoneApiTestsBắt buộc test:create zone success create zone duplicate fail update zone success deactivate zone success/fail theo policy 12.2. tests/integration/source-origin/SourceOriginApiTestsBắt buộc test:create origin với zone hợp lệ create origin với zone không tồn tại → fail create origin với zone inactive → fail 12.3. tests/integration/source-origin/SourceOriginEvidenceTestsBắt buộc test:attach evidence success attach evidence vào source origin không tồn tại → fail 12.4. tests/integration/source-origin/SourceOriginVerificationTestsBắt buộc test:verify success verify tạo record mới, không overwrite record cũ đọc detail thấy đúng zone + evidences + verification Các điểm này trùng hoàn toàn với test matrix của FILE 04 và checklist CODE 01 trong FILE 05.13. Smoke test phase 01Tạo file:tests/smoke/code01_source_origin_smoke.md Nội dung smoke tối thiểu:tạo source zone ZONE_BENTRE_001 tạo lại cùng zone_code → fail unique tạo source origin với source_zone_id hợp lệ → success attach evidence → success verify source origin → success đọc lại detail source origin → thấy đủ: zone evidence verification latest / history Đây chính là chuỗi smoke tối thiểu của phase 01. 14. Done gate của CODE 01ĐẠT khiđủ 8 bảng foundation / governance đủ 4 bảng source origin route family đúng namespace create/query/update/deactivate/attach evidence/verify chạy được test unique/FK/bind pass không có owner mơ hồ phase 02 bind được source_zone_id và source_origin_id xuống intake. CHƯA ĐẠT khizone_code không unique source origin không bind chuẩn về zone evidence / verification bị nhét sai owner không có idempotency cho create/verify route family sai namespace phase sau vẫn phải tự đoán source origin. 15.
