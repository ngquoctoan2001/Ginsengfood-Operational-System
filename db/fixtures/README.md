# Database Fixtures

Dev/test/smoke-only fixture area.

Allowed examples:

- fake GTIN and GS1 identities for local tests;
- fake MISA mapping and dry-run integration data;
- source origin samples;
- batch smoke fixtures.

Fixtures here must not be treated as production seed data. Production or canonical seed data belongs in `db/seeds/` and must trace back to `docs/software-specs/`.
