# 03 Screen Catalog

## 1. Mục tiêu

Catalog này là hợp đồng giữa BA/SA, FE, BE và QA cho từng màn hình. Mỗi screen phải biết rõ route, role, API source, field, table, filter, action, validation, empty/error state, permission, workflow và test case liên quan.

## 2. Quy tắc chung

- `route` là route UI mục tiêu, không phải API route.
- `data source API` phải dùng route family trong `api/02_API_ENDPOINT_CATALOG.md`.
- `actions` chỉ là hành động UI; backend vẫn là nơi enforce permission/state/validation.
- Mọi command tạo giao dịch hoặc đổi state phải gửi `Idempotency-Key`.
- Public trace chỉ dùng `/api/public/trace/{qrCode}` và không expose supplier/personnel/cost/QC defect/loss/MISA.
