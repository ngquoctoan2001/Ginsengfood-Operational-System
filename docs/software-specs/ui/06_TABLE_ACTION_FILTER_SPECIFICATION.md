# 06 Table Action Filter Specification

## 1. Mục tiêu

Tài liệu này chuẩn hóa table columns, filter, row action, bulk action và permission cho các list screen. Quy tắc filter/sort/pagination phải đồng bộ với `api/07_API_PAGINATION_FILTER_SORT_SPEC.md`.

## 2. Quy tắc table chung

| Hạng mục     | Quy tắc                                                                                                         |
| ------------ | --------------------------------------------------------------------------------------------------------------- |
| Pagination   | Dùng `page`, `page_size`; mặc định theo API convention.                                                         |
| Sort         | Chỉ sort field được API cho phép; default sort theo `created_at desc` hoặc field nghiệp vụ rõ ràng.             |
| Filter       | Filter UI phải map 1-1 sang query param API; không tự filter dữ liệu phân trang ở client nếu ảnh hưởng kết quả. |
| Row action   | Chỉ hiển thị khi user có permission và record state cho phép.                                                   |
| Bulk action  | Chỉ dùng khi API hỗ trợ bulk và idempotency rõ ràng; nếu chưa có API, đánh dấu OWNER DECISION NEEDED.           |
| Export       | Export phải dùng endpoint riêng hoặc cùng filter query; không export dữ liệu public-forbidden.                  |
| Status badge | Dùng enum chuẩn, không dùng text tự do.                                                                         |
| Empty state  | Ghi rõ "chưa có dữ liệu" theo ngữ cảnh, không gợi ý thao tác user không có quyền.                               |
| Error state  | Hiển thị error code/message/correlation_id nếu API trả về.                                                      |

## 4. Row Action Rules

| Action type         | UI condition                                                                         | Backend requirement                                                           | Idempotency                                                                                   |
| ------------------- | ------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `create`            | User has write permission                                                            | Validate uniqueness and required fields                                       | Có nếu tạo transaction/stateful record                                                        |
| `edit`              | Record editable state and user has write permission                                  | Validate optimistic concurrency if supported                                  | Không bắt buộc nếu PATCH idempotent by resource state; khuyến nghị có key cho critical change |
| `approve`           | Pending approval and approver role                                                   | Approver cannot approve own change nếu policy yêu cầu [OWNER DECISION NEEDED] | Có                                                                                            |
| `reject`            | Pending approval                                                                     | Reason required                                                               | Có                                                                                            |
| `cancel`            | State cancellable                                                                    | Reason required if document already submitted                                 | Có                                                                                            |
| `hold`              | Target not already held                                                              | Reason required; audit                                                        | Có                                                                                            |
| `release_hold`      | Target held and approval if required                                                 | Reason required; audit                                                        | Có                                                                                            |
| `mark_ready`        | Raw lot `QC_PASS`, not held/rejected/quarantined, not already `READY_FOR_PRODUCTION` | Creates readiness transition/audit; target `READY_FOR_PRODUCTION`             | Có                                                                                            |
| `issue_material`    | Material issue OPEN/APPROVED                                                         | Lot `READY_FOR_PRODUCTION`; sufficient balance; snapshot line                 | Có                                                                                            |
| `confirm_receipt`   | Receipt pending                                                                      | Quantity/variance validation                                                  | Có                                                                                            |
| `release_batch`     | QC_PASS and not RELEASED                                                             | Release action creates release record                                         | Có                                                                                            |
| `warehouse_receive` | Batch RELEASED                                                                       | Creates inventory ledger and balance projection                               | Có                                                                                            |
| `qr_void`           | QR not already VOID                                                                  | Reason required                                                               | Có                                                                                            |
| `qr_reprint`        | QR PRINTED/FAILED according lifecycle                                                | Reason required; reprint audit                                                | Có                                                                                            |
| `misa_retry`        | Sync job FAILED/RETRYABLE                                                            | Through integration layer only                                                | Có                                                                                            |

## 6. Empty/Error State Chuẩn Cho Table

| State                    | UI behavior                                                                                               |
| ------------------------ | --------------------------------------------------------------------------------------------------------- |
| First empty              | Nếu user có create permission, hiển thị primary create action; nếu không, chỉ hiển thị message read-only. |
| Filter empty             | Hiển thị "Không có kết quả theo bộ lọc hiện tại" và nút clear filters.                                    |
| Permission denied        | Không render table data; hiển thị message thiếu quyền.                                                    |
| API error                | Hiển thị error message, error code, correlation id và retry.                                              |
| Stale data after command | Reload row/list; nếu conflict, show stale state message.                                                  |
| Partial data             | Dùng warning banner, đặc biệt trace/genealogy/recall impact.                                              |
