## 3. Check Constraints

| constraint_id                    | Table                                                  | Rule                                                                                                                                                                                                                                                                                                                                                               |
| -------------------------------- | ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| CK-OPERATIONAL-RECIPE-VERSION    | `op_production_recipe`                                 | `formula_version` phải thuộc registry/version policy được phê duyệt cho operational status; G0/research/baseline token không được `APPROVED` hoặc `ACTIVE_OPERATIONAL`.                                                                                                                                                                                            |
| CK-FORMULA-KIND                  | `op_production_recipe`                                 | `formula_kind IN ('PILOT_PERCENT_BASED','FIXED_QUANTITY_BATCH')`; G1 mặc định `PILOT_PERCENT_BASED`, G2 mặc định `FIXED_QUANTITY_BATCH` (validate ở service, không hard-code DB).                                                                                                                                                                                  |
| CK-RECIPE-ANCHOR-NOTNULL         | `op_production_recipe`                                 | `formula_kind = 'PILOT_PERCENT_BASED'` ⇒ `anchor_ingredient_id`, `anchor_baseline_quantity`, `anchor_uom_code`, `anchor_ratio_percent` NOT NULL và > 0; `formula_kind = 'FIXED_QUANTITY_BATCH'` ⇒ tất cả anchor fields NULL.                                                                                                                                       |
| CK-RECIPE-LINE-MODE              | `op_recipe_ingredient`                                 | PILOT recipe: anchor line có `is_anchor=true` + `ratio_percent NOT NULL > 0`; non-anchor PILOT line có `ratio_percent NOT NULL > 0` và `quantity_per_batch_400 NULL`. FIXED recipe: mọi line `is_anchor=false`, `quantity_per_batch_400 NOT NULL > 0`, `ratio_percent` không bắt buộc. Implement bằng trigger/service guard nếu DB không hỗ trợ cross-table CHECK. |
| CK-RECIPE-RATIO-SUM              | `op_recipe_ingredient`                                 | PILOT recipe: `SUM(ratio_percent) ∈ [99.95, 100.05]` per `recipe_id`. Implement bằng trigger sau insert/update line hoặc service-level guard tại approve/activate.                                                                                                                                                                                                 |
| CK-PO-FORMULA-KIND               | `op_production_order`                                  | `formula_kind_snapshot IN ('PILOT_PERCENT_BASED','FIXED_QUANTITY_BATCH')`; PILOT ⇒ anchor inputs NOT NULL > 0 và `batch_size NULL`; FIXED ⇒ `batch_size NOT NULL > 0` và anchor inputs NULL.                                                                                                                                                                       |
| CK-POI-SNAPSHOT-BASIS            | `op_production_order_item`                             | `snapshot_basis IN ('PILOT_RATIO_OF_ANCHOR','FIXED_PER_BATCH_N')`; PILOT PO ⇒ `snapshot_basis='PILOT_RATIO_OF_ANCHOR'`; FIXED PO ⇒ `snapshot_basis='FIXED_PER_BATCH_N'`; `snapshot_quantity > 0`.                                                                                                                                                                  |
| CK-RECIPE-GROUP                  | `ref_recipe_line_group`                                | group_code in 4 required values.                                                                                                                                                                                                                                                                                                                                   |
| CK-PROCUREMENT-TYPE              | `op_raw_material_lot`                                  | `procurement_type IN ('SELF_GROWN','PURCHASED')`.                                                                                                                                                                                                                                                                                                                  |
| CK-PROCUREMENT-FIELDS            | `op_raw_material_lot`                                  | `SELF_GROWN` requires `source_origin_id` and null `supplier_id`; `PURCHASED` requires `supplier_id` and null `source_origin_id`.                                                                                                                                                                                                                                   |
| CK-RAW-LOT-STATUS                | `op_raw_material_lot`                                  | `lot_status IN ('CREATED','IN_QC','ON_HOLD','REJECTED','READY_FOR_PRODUCTION','CONSUMED','EXPIRED','QUARANTINED')`; `READY_FOR_PRODUCTION` is distinct from `lot_qc_status = 'QC_PASS'`.                                                                                                                                                                           |
| CK-MATERIAL-ISSUE-LOT-READY      | `op_material_issue_line`                               | Issue line insert/update must verify referenced raw lot has `lot_status = 'READY_FOR_PRODUCTION'`; implement as service guard plus DB trigger/deferred constraint if cross-table checks are supported.                                                                                                                                                             |
| CK-QC-STATUS                     | QC tables                                              | Only `PENDING_QC`, `QC_PASS`, `QC_HOLD`, `QC_REJECT`.                                                                                                                                                                                                                                                                                                              |
| CK-BATCH-STATUS                  | `op_batch`                                             | Only `CREATED`, `IN_PROCESS`, `PACKAGED`, `QC_PENDING`, `QC_PASS`, `QC_HOLD`, `QC_REJECT`, `RELEASED`, `BLOCKED`, `CLOSED`.                                                                                                                                                                                                                                        |
| CK-RECALL-STATUS                 | `op_recall_case`                                       | Includes `CLOSED_WITH_RESIDUAL_RISK`; when selected, `residual_note` is required.                                                                                                                                                                                                                                                                                  |
| CK-EVIDENCE-TYPE                 | `op_source_origin_evidence`, `op_recall_capa_evidence` | `evidence_type IN ('FIELD_PHOTO','FIELD_VIDEO','CERTIFICATE_DOC','LAB_REPORT','CONTRACT_DOC','OTHER')`.                                                                                                                                                                                                                                                            |
| CK-EVIDENCE-MIME-SIZE            | `op_source_origin_evidence`, `op_recall_capa_evidence` | `mime_type IN ('image/jpeg','image/png','image/webp','video/mp4','video/quicktime')`; image MIME types require `file_size_bytes <= 10*1024*1024`; video MIME types require `file_size_bytes <= 100*1024*1024`; all evidence requires `file_size_bytes > 0`.                                                                                                        |
| CK-EVIDENCE-SCAN-STATUS          | `op_source_origin_evidence`, `op_recall_capa_evidence` | `scan_status IN ('PENDING_SCAN','CLEAN','INFECTED','SCAN_FAILED')`; source verification and CAPA/recall close gates accept only `CLEAN`. Dev/test may use mock/dev-skip scanner mode to produce `CLEAN`; production must use a real AV/malware scanner.                                                                                                            |
| CK-WAREHOUSE-RECEIPT-RELEASED    | `op_warehouse_receipt`                                 | Receipt confirmation must reference an `op_batch_release` row for the same batch with `release_status = 'APPROVED_RELEASED'`; implement with trigger/deferred validation if FK cannot include the status predicate.                                                                                                                                                |
| CK-PROCESS-STEP                  | `op_production_process_event`                          | Only `PREPROCESSING`, `FREEZING`, `FREEZE_DRYING`.                                                                                                                                                                                                                                                                                                                 |
| CK-QR-STATUS                     | `op_qr_registry`                                       | Only `GENERATED`, `QUEUED`, `PRINTED`, `FAILED`, `VOID`, `REPRINTED`.                                                                                                                                                                                                                                                                                              |
| CK-WAREHOUSE-TYPE                | `op_warehouse`                                         | Only `RAW_MATERIAL`, `FINISHED_GOODS`.                                                                                                                                                                                                                                                                                                                             |
| CK-QTY-POSITIVE                  | Quantity columns                                       | `quantity > 0` except reversal/adjustment rows where signed quantity is explicit.                                                                                                                                                                                                                                                                                  |
| CK-TRADE-ITEM-IDENTIFIER-FIXTURE | `op_trade_item_gtin`                                   | DB stores and constrains the `is_test_fixture` flag và `identifier_type` (bao gồm `TEST_FIXTURE`); production commercial print block là service-level policy dùng cờ `is_test_fixture` hoặc `identifier_type='TEST_FIXTURE'`, không fallback theo SKU code. Tên bảng V2 locked theo ADR-025.                                                                       |

## 4. Append-Only Guards

| guard_id           | Table                                                  | Guard                                                                                                                                                                                                                                       |
| ------------------ | ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| AO-AUDIT           | `audit_log`                                            | No UPDATE/DELETE.                                                                                                                                                                                                                           |
| AO-STATE           | `state_transition_log`                                 | No UPDATE/DELETE.                                                                                                                                                                                                                           |
| AO-LEDGER          | `op_inventory_ledger`                                  | No UPDATE/DELETE after insert; correction via reversal/adjustment.                                                                                                                                                                          |
| AO-QR-HISTORY      | `op_qr_state_history`                                  | No UPDATE/DELETE.                                                                                                                                                                                                                           |
| AO-PRINT-LOG       | `op_print_log`                                         | No UPDATE/DELETE.                                                                                                                                                                                                                           |
| AO-RECALL-SNAPSHOT | `op_recall_exposure_snapshot`                          | No UPDATE/DELETE; new `snapshot_version` for rerun.                                                                                                                                                                                         |
| AO-EVIDENCE        | `op_source_origin_evidence`, `op_recall_capa_evidence` | No UPDATE/DELETE except controlled scan-status transition from `PENDING_SCAN` to terminal scan result; metadata corrections require a new evidence row.                                                                                     |
| AO-EVENT           | `event_store`                                          | No UPDATE/DELETE; for `outbox_event`, only `status`, `retry_count`, `next_retry_at`, `dispatched_at` may update after insert. `payload`, `event_type`, `event_version`, `aggregate_type`, `aggregate_id` and `privacy_class` are immutable. |

Implementation option: DB trigger `prevent_update_delete_append_only()` on append-only tables plus restricted DB role permissions.
