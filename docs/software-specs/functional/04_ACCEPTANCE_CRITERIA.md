## 2. QA Notes

- Negative tests are mandatory for every hard lock: research baseline token forbidden, non-verified source, lot `QC_PASS` but not `READY_FOR_PRODUCTION`, outside-snapshot material, QC pass not release, warehouse before release, QR void/failed public trace, public field leakage, missing MISA mapping.
- For docs generated before implementation, API/table names are target contracts. Implementation validation must be added later in API/database/testing specs.
- Owner-decision items must be tested as configurable/pending until owner chốt: trace SLA, public i18n, recall SLA, MISA credential, printer driver, retention/RPO/RTO.
