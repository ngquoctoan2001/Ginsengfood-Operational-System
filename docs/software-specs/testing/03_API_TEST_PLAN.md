# 03 - API Test Plan

## 1. Mục tiêu

API test plan kiểm tra endpoint contract trong [api/02_API_ENDPOINT_CATALOG.md](../api/02_API_ENDPOINT_CATALOG.md): method/path, auth, permission, request/response, error code, idempotency, state transition, audit và dữ liệu liên quan.

## 2. API Test Principles

| principle_id | Principle                                                                                                                               |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| API-TP-001   | Mỗi endpoint command P0 phải có happy path, validation negative, permission negative và idempotency test nếu catalog đánh dấu required. |
| API-TP-002   | API test phải assert cả response và side effect: table target, audit, state transition, outbox/ledger nếu có.                           |
| API-TP-003   | Public API test dùng whitelist response keys; không reuse assertion từ admin/internal trace.                                            |
| API-TP-004   | Error response phải có code ổn định, message, details/correlation id nếu convention yêu cầu.                                            |

- API P0 tests cover all hard locks in [01_TEST_STRATEGY.md](01_TEST_STRATEGY.md).

- Public API tests assert denylist/whitelist explicitly.
- Command tests assert idempotency where required.
