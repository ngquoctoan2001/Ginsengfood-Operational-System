# 04 - UI Test Plan

## 1. Mục tiêu

UI test plan kiểm tra screen contract trong [ui/03_SCREEN_CATALOG.md](../ui/03_SCREEN_CATALOG.md): route, role, API source, form fields, table columns, filters, actions, validation, empty state, error state và permission-aware behavior.

## 2. UI Test Principles

| principle_id | Principle                                                                                                   |
| ------------ | ----------------------------------------------------------------------------------------------------------- |
| UI-TP-001    | UI chỉ là lớp trải nghiệm; mọi action nhạy cảm phải có API permission test tương ứng.                       |
| UI-TP-002    | Mỗi màn hình P0 phải test render, load data, form validation, empty state, error state và permission state. |
| UI-TP-003    | Public trace UI chỉ render field whitelist; không ẩn bằng CSS nếu API vẫn trả private field.                |
| UI-TP-004    | E2E/UI automation nên dùng stable `data-testid` theo screen/action, không phụ thuộc text dễ thay đổi.       |

## 4. Screen State Checklist

| screen_group         | Must test states                                                                                                                      |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| Master data          | loading, empty, duplicate code, inactive dependency, permission denied                                                                |
| Recipe/PO            | missing active recipe, invalid group, forbidden operational version token, snapshot read-only                                         |
| Material             | `QC_PASS` chưa mark-ready, `READY_FOR_PRODUCTION`, lot not ready, insufficient balance, duplicate idempotency replay, variance reason |
| QC/Release/Warehouse | QC hold/reject reason, QC pass not released, release hold active, batch not released                                                  |
| QR/Public trace      | generated/queued/printed/failed/void/reprinted, public-safe invalid, forbidden field policy                                           |
| Recall/MISA          | open/impact/hold/close blocked, missing mapping, retry/reconcile failed                                                               |

## 5. Done Gate

- Mỗi UI test P0 map được về `screen_id` và `REQ-*`.
- UI public trace tests không chỉ kiểm giao diện mà assert response field policy.
- Permission UI tests đi kèm API direct-call negative test trong API pack.
