> Mục đích: xác định ai được tạo, duyệt, xác nhận, release và override từng nghiệp vụ. Permission phải được enforce ở backend, UI chỉ là lớp hỗ trợ.

### 1.2 Permission Namespace Policy

Hai namespace `action_code` đang dùng song song — đây là policy chính thức:

| Namespace         | Pattern         | Dùng cho                                                                          | Ví dụ                                                                                           |
| ----------------- | --------------- | --------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| **UPPER_SNAKE**   | `DOMAIN_ACTION` | Tất cả admin/internal operations trên `/api/admin/*` trừ M03A supplier management | `RAW_INTAKE_RECEIVE`, `PRODUCTION_ORDER_APPROVE`, `BATCH_RELEASE_APPROVE`, `QC_INSPECTION_SIGN` |
| **dot namespace** | `domain.action` | M03A supplier management admin operations trên `/api/admin/suppliers/*`           | `supplier.read`, `supplier.suspend`, `supplier.user.manage`                                     |
| **dot namespace** | `supplier.*`    | Supplier portal operations trên `/api/supplier/*`                                 | `supplier.receipt.read`, `supplier.receipt.submit`, `supplier.self.read`                        |

**Quy tắc áp dụng:**

- `auth_permission.action_code` trong database seed dùng đúng namespace theo bảng trên.
- UI `required_permission` trong sidebar/screen spec là hint để gate access; phải khớp với `action_code` trong `data/csv/roles_permissions.csv`.
- Backend middleware enforce `action_code` từ `auth_permission` — không enforce từ UI hint.
- Không tạo alias hoặc synonym giữa hai namespace trong cùng domain.

Legend: `C` create, `U` update draft, `S` submit, `A` approve, `Rj` reject, `X` execute/confirm, `Rel` release, `H` hold/halt, `Can` cancel, `Corr` correction, `Ovr` override, `View` read.

## 4. Permission Rules

| permission_rule_id | Rule                                                                                                                                                                     | Validation                                                                                                          | Exception                                                                     |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| PR-001             | Không role nào được tự bypass backend permission bằng UI.                                                                                                                | API luôn kiểm `permission/action`.                                                                                  | Không có exception.                                                           |
| PR-002             | Read-only role không gọi được command endpoint.                                                                                                                          | `R-AUDITOR` chỉ view/export nếu policy cho phép.                                                                    | Export sensitive cần permission riêng.                                        |
| PR-003             | Separation of duty áp dụng cho recipe activation, release, inventory adjustment lớn, override nếu owner chốt.                                                            | Submitter và approver khác user khi policy yêu cầu.                                                                 | Owner có thể chốt threshold sau.                                              |
| PR-004             | Override không được sửa ledger/audit/snapshot in-place.                                                                                                                  | Override chỉ tạo action log/correction/reversal.                                                                    | Không có exception.                                                           |
| PR-005             | Public trace API không dùng admin permission context.                                                                                                                    | Public route chỉ dùng QR token/code và field policy.                                                                | Không trả private field dù user admin gọi public route.                       |
| PR-006             | QC correction do `R-QC-RAW`/`R-QC-PROD` đề xuất phải qua `R-QA-REL` hoặc policy approver trước khi có hiệu lực.                                                          | Matrix có thể cho role QC tạo correction request, nhưng backend phải enforce approval dependency.                   | Emergency correction vẫn đi qua override/audit, không sửa signed QC in-place. |
| PR-007             | `RAW_LOT_MARK_READY` là permission riêng với `RAW_QC_SIGN`.                                                                                                              | User ký QC không tự động có quyền mark-ready nếu không được cấp action.                                             | `R-OPS-MGR` có thể override có audit.                                         |
| PR-008             | `R-SUPPLIER` chỉ được gọi route `/api/supplier/*`; backend bắt buộc resolve `supplier_id` từ `op_supplier_user_link` và scope mọi query/command theo đúng `supplier_id`. | API trả `403 SUPPLIER_SCOPE_VIOLATION` khi token cố đọc/ghi intake/evidence của supplier khác; không leak metadata. | Không có exception.                                                           |
| PR-009             | `R-SUPPLIER` không được gọi bất kỳ route `/api/admin/*` hay `/api/mobile/*`.                                                                                             | Backend kiểm role + audience claim trước route handler.                                                             | Không có exception.                                                           |
| PR-010             | Supplier user chỉ submit/confirm intake với ingredient nằm trong `op_supplier_ingredient` của `supplier_id` đó (HL-SUP-002 / HL-SUP-013).                                | API trả `SUPPLIER_INGREDIENT_NOT_ALLOWED` khi vi phạm.                                                              | Allowlist phải approved + status `ACTIVE` (UI-VAL-SUP-006).                   |
| PR-011             | Mật khẩu supplier phải tuân `HL-SUP-008` (độ dài tối thiểu, complexity, hash bcrypt/argon2, lockout; không lưu plaintext).                                               | Backend reject password yếu; reset password ghi audit `supplier.user.reset_password`.                               | Không lưu plaintext; không log password.                                      |
| PR-012             | Supplier `SUSPENDED` block toàn bộ login + command (`UI-VAL-SUP-001`); reactivate cần `R-OPS-MGR` (`AP-SUP-CONFIRM` policy không áp dụng).                               | Backend kiểm `supplier.status` trước mọi request.                                                                   | Audit suspend/reactivate.                                                     |
