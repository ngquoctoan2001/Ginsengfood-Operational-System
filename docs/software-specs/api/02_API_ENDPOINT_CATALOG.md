## 2. Public Trace DTO Contract

`GET /api/public/trace/{qrCode}` and `GET /api/admin/trace/public-preview/{qrCode}` must return `PublicTracePublicResponse`, not `InternalTraceResponse`, `GenealogyTraceResponse`, ORM entities, or any admin DTO. OpenAPI must define this schema with `additionalProperties = false`.

Allowed fields are exactly the active public whitelist in `docs/software-specs/data/csv/public_trace_policy.csv`:

| DTO path                    | Source policy field     | Notes                                                      |
| --------------------------- | ----------------------- | ---------------------------------------------------------- |
| `schemaVersion`             | system constant         | Must be `public-trace.v1`; not a DB/internal identifier.   |
| `traceStatus`               | public resolver status  | Public-safe enum: `VALID`, `NOT_PUBLIC`, `INVALID_QR`.     |
| `qr.qrStatusPublic`         | `qr_status_public`      | Do not expose raw QR lifecycle reason for `VOID`/`FAILED`. |
| `product.productName`       | `product_name`          | Public display name only.                                  |
| `batch.batchPublicCode`     | `batch_public_code`     | Public-safe batch code, not `internal_batch_id`.           |
| `batch.releasePublicStatus` | `release_public_status` | Public-safe release status only.                           |
| `source.sourceZoneName`     | `source_zone_name`      | Public approved zone name.                                 |
| `source.province`           | `province`              | Public approved province.                                  |
| `source.ward`               | `ward`                  | Public approved ward.                                      |
| `source.addressDetail`      | `address_detail`        | Public approved address text.                              |

Canonical response shape:

```json
{
  "schemaVersion": "public-trace.v1",
  "traceStatus": "VALID",
  "qr": {
    "qrStatusPublic": "PRINTED"
  },
  "product": {
    "productName": "Cháo Sâm - Diêm mạch & Hạt sen"
  },
  "batch": {
    "batchPublicCode": "BATCH-PUBLIC-20260427-001",
    "releasePublicStatus": "RELEASED"
  },
  "source": {
    "sourceZoneName": "Vùng trồng Sâm Savigin Lâm Đồng",
    "province": "Lâm Đồng",
    "ward": "Xã ...",
    "addressDetail": "Thông tin public đã duyệt"
  }
}
```

Forbidden fields in public trace include `supplier_id`, `supplier_name`, `operator_user_id`, `qc_defect_detail`, `loss_quantity`, `cost_amount`, `misa_external_id`, `internal_batch_id`, `raw_material_lot_internal_code`, evidence URI/hash/original filename/storage path/scan payload, personnel names, raw ledger ids, formula/costing, and any customer/order/shipment data. `producedAt` and `releasedAt` are not public fields until owner adds explicit rows to `public_trace_policy.csv`.

Invalid, void, failed, or non-public QR must return only public-safe status/error envelope; do not leak internal QR state reason, supplier identity, MISA mapping, QC defect, evidence scan detail, or trace gap internals.

## 4. PF-02 Production Config/API Closure

| area             | API contract rule                                                                                                                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| GTIN/GS1         | Identifier APIs must expose `is_test_fixture`; production commercial print rejects fixture identifiers. PACKET has no QR/trace standalone; BOX/CARTON identifiers drive print/trace logistics. |
| MISA AMIS        | MISA APIs manage sync/mapping/reconcile state only. Tenant, endpoint and credential values are deploy-time config/secret refs and must not be returned by admin APIs.                          |
| Printer/device   | Device callbacks use HMAC-authenticated `PrintJobCallbackRequest` and idempotency. Callback may update print/QR technical status only; it cannot mark QC, release, inventory or batch state.   |
| Evidence storage | Evidence create APIs accept metadata/upload reference, not DB blob. Response may include evidence id/status, never storage secret, raw server path, malware payload or scan engine internals.  |
| Notification     | Recall APIs store `notification_job_id`/outbox status references only. Delivery API/gateway contract belongs to external sales/notification system.                                            |
| Rate limits      | Public trace, admin commands, supplier/PWA submit, MISA callback and device callback route families must be rate-limitable by config.                                                          |
