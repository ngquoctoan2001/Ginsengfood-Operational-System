# 08 Smoke Workflow

## 1. Mục tiêu

Smoke workflow này là kịch bản E2E tối thiểu có thể dùng trực tiếp để viết test cho flow vận hành chính. Kịch bản đi từ source origin tới raw material, production, issue/receipt, batch, packaging/QR, QC/release, warehouse, trace, recall và MISA sync.

## 2. Smoke Dataset

| data_key | Giá trị gợi ý | Ghi chú |
|---|---|---|
| `source_zone_code` | `SZ_SMOKE_001` | Source zone test |
| `source_origin_code` | `SO_SMOKE_001` | Phải được verify |

## 5. Negative Smoke Steps

| negative_id | Scenario | Setup | Action | Expected result | Test assertion |
|---|---|---|---|---|---|
| SMK-N001 | Intake with unverified source | Source origin `SUBMITTED` | Create raw intake requiring verified source | API rejects | Error `SOURCE_ORIGIN_NOT_VERIFIED` |
| SMK-N002 | Issue raw lot not ready for production | Raw lot `PENDING_QC`/`QC_HOLD` or `QC_PASS` but not `READY_FOR_PRODUCTION` | Execute material issue | API rejects | Error `LOT_NOT_READY_FOR_PRODUCTION`; no ledger |
| SMK-N003 | Issue material outside snapshot | Material request line not in PO snapshot | Submit/execute request | API rejects | Error `OUTSIDE_SNAPSHOT_MATERIAL` |
| SMK-N004 | Duplicate issue retry | Same issue execute payload and same idempotency key | Submit twice | Second response is replay/same outcome | Only one ledger decrement |
| SMK-N005 | Receipt variance without reason | `received_qty != issued_qty` and no reason | Confirm receipt | API rejects | Error `VARIANCE_REASON_REQUIRED` |
| SMK-N006 | Warehouse receipt before release | Batch QC_PASS but no release | Confirm warehouse receipt | API rejects | Error `BATCH_NOT_RELEASED` |
| SMK-N007 | Public trace leakage | Public trace response contains forbidden field | Preview/public resolve | Block preview or suppress field | No supplier/personnel/cost/QC defect/loss/MISA visible |
| SMK-N008 | MISA mapping missing | Sync event without mapping | Dispatch/retry sync | `FAILED_NEEDS_REVIEW` | Mapping/reconcile action visible |
| SMK-N009 | QR void public trace | QR state `VOID` | Public trace resolve | Public-safe invalid/void response | No internal void reason leaked |
| SMK-N010 | Recall close with open CAPA | Recall case has open recovery/CAPA | Close recall | API rejects | Error `RECOVERY_OPEN` or `CAPA_REQUIRED` |

## 7. E2E Smoke Acceptance Criteria

| ac_id | Acceptance criteria |
|---|---|
| SMK-AC-001 | Full chain completes from verified source origin to warehouse receipt without manual DB intervention. |
| SMK-AC-002 | PO snapshot contains active G1 recipe lines and is immutable after creation/start. |
| SMK-AC-003 | Raw lot must pass QC and then be marked `READY_FOR_PRODUCTION` before material issue. |
| SMK-AC-004 | Material issue posts exactly one raw inventory decrement ledger entry, even after idempotent retry. |
| SMK-AC-005 | Material receipt confirmation does not post a second decrement ledger entry. |
| SMK-AC-006 | Batch cannot be warehouse-received until release record is approved. |
| SMK-AC-007 | Finished-goods warehouse receipt posts ledger and updates lot balance projection. |
| SMK-AC-008 | Internal trace shows source/raw lot/issue/batch/QR/warehouse chain. |
| SMK-AC-009 | Public trace returns only whitelisted fields and no forbidden internal fields. |
| SMK-AC-010 | MISA sync event is visible with terminal status `SYNCED`, `FAILED_NEEDS_REVIEW`, or `RECONCILED` according test environment. |
| SMK-AC-011 | Recall extension can apply hold/sale lock and close only after recovery/CAPA completion. |
