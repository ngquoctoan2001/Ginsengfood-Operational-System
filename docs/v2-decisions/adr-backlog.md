## 3. ADR Creation Rule

Create a full ADR file when a backlog item changes one of these surfaces:

| Surface                     | ADR required when                                                                            |
| --------------------------- | -------------------------------------------------------------------------------------------- |
| API/OpenAPI/frontend client | Route taxonomy, envelope, DTO, error shape, auth/permission or idempotency behavior changes  |
| Database/migration/seed     | Table ownership, enum, constraint, index, migration order, snapshot or seed baseline changes |
| Workflow/state machine      | QC/release, lot readiness, inventory ledger, recall or QR/print lifecycle changes            |
| Integration                 | MISA, printer/device, public trace or external reference contract changes                    |
| Operations                  | Backup/restore, retention/archive, observability, deployment, smoke or release gates change  |
